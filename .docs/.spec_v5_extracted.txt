1. Leitprinzip & Architektur
1.1 Credo
Effizienz vor Bürokratie. WIMUS ERP ist eine Schaltzentrale – keine Eigenentwicklung von Funktionalität die es als fertiges Open-Source-Modul gibt.
WIMUS ERP = Cockpit/Schaltzentrale (Next.js + Supabase)
Fertige Open-Source-Module werden über API angebunden – nie nachgebaut
Pflichtfelder auf Minimum · Standardwerte vorausfüllen · max. 2 Klicks
KI übernimmt Erfassungsarbeit (OCR, Vorschläge) → Mensch bestätigt, korrigiert, entscheidet
Mobile-first von Anfang an – jede Komponente responsive
Lieber 80% automatisch korrekt als 100% manuell perfekt

9. Sicherheitskonzept & DSGVO
Sofort / MVP-Pflicht
RLS in Supabase von Tag 1 (mandant_id-Isolation auf Datenbankebene)
MFA Pflicht alle internen User (Supabase Auth)
Service-Role-Key nur serverseitig (nie im Frontend/Browser)
Keine Secrets im Code/Repo (Coolify Environment Variables)
TLS überall (Traefik SSL) · SSH Key-only, kein Passwort-Login, Fail2Ban
Laufend
Webhook-Signatur-Verifizierung (Beds24, GreenAPI, Forminator)
Sensible Felder verschlüsselt (Ausweisnummern, Bankdaten – Supabase Vault/pgcrypto)
3-2-1 Backup (3 Kopien, 2 Medien, 1 offsite) + regelmäßiger Restore-Test!
Audit-Log unveränderlich · GoBD-Paket (GDPdU/IDEA-Export, Verfahrensdokumentation §147 AO)
DSGVO
AVV mit ALLEN Dienstleistern: Anthropic (Claude API!), Caya, GreenAPI, seven.io, Retell AI, Beds24
Löschkonzept: Bewerber 6 Monate, Verträge 3J nach Ende, Buchungsbelege 10J (GoBD)
dsgvo_datenweitergabe Pflichtfeld je Kontakt · Meldeschein Reisepass → Löschkonzept
Incident Response: 72h-Meldepflicht an Aufsichtsbehörde bei Datenpanne!
Steuerberater-Zugang: eigene Rolle, nur Buchungen/Belege – NICHT Mieterdaten
